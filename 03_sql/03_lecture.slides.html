Databases and SQL¶

Learning Objectives¶

  1. Retrieve rows from tables using SELECT, FROM, and LIMIT clauses
  2. Demonstrate an understanding of primary and foreign keys
  3. Utilize aggregate functions to summarize data in tables
  4. Filter rows from queries using WHERE, IN, LIKE, AND and OR clauses
  5. Group, summarize, and sort results from queries using GROUP BY, HAVING, and ORDER BY clauses
  6. Join tables using a LEFT JOIN

Databases¶

A database is an organized collection of data stored electronically so it can be easily accessed, managed, and updated.

In many databases, data is stored in tables made up of rows and columns. Each row represents a single record, and each column represents an attribute of that record.

In [22]:

import sqlite3
from pprint import pprint

# Creating a connectiong to the database
conn = sqlite3.connect("../data/movies.sqlite")

# The cursor will allow us to query the database
cur = conn.cursor()

Selecting Columns and Rows¶

SELECT, FROM, and LIMIT¶

The SELECT statement specifies which columns should be returned in the query results.

The FROM clause specifies the table that the data should be retrieved from.

The LIMIT clause restricts the number of rows returned by the query.

Keys¶

A primary key is a column (or set of columns) that uniquely identifies each row in a table. No two rows can have the same primary key value, and the value cannot be NULL.

A foreign key is a column in one table that references the primary key in another table. Foreign keys create relationships between tables and help maintain consistent and connected data.

DISTINCT¶

The DISTINCT keyword is used in a query to return only unique values from a column or set of columns.

When DISTINCT is used, duplicate rows in the result are removed so that each value appears only once.

In [26]:
res = cur.execute("""
    SELECT DISTINCT director_id FROM movies
    LIMIT 5
""")

res.fetchall()
Out[26]:
[(4762,), (4763,), (4764,), (4765,), (4766,)]

Aggregate Functions¶

Aggregate Functions¶

An aggregate function performs a calculation on multiple rows and returns a single summarized value.

Common aggregate functions include COUNT, SUM, AVG, MIN, and MAX, which are used to summarize data in a table.

COUNT¶

The COUNT function returns the number of rows that match a query.

It is commonly used to determine how many records exist in a table or how many rows satisfy a particular condition.

In [27]:
# Gets all movies
res = cur.execute("""
    SELECT COUNT(*), COUNT(DISTINCT director_id) FROM movies
""")

print(res.fetchall())

[(4773, 2349)]

AVG and SUM¶

The AVG function calculates the average (mean) value of a numeric column across the rows returned by a query.

The SUM function adds together the values in a numeric column to return the total.

In [28]:
res = cur.execute("""
    SELECT AVG(vote_average) FROM movies
""")

print(res.fetchall())

res = cur.execute("""
    SELECT SUM(budget) FROM movies
""")

print(res.fetchall())

[(6.109176618478943,)]
[(139489076520,)]

MIN and MAX¶

The MIN function returns the smallest value in a column.

The MAX function returns the largest value in a column.

In [29]:
res = cur.execute("""
    SELECT MAX(vote_count), title FROM movies
""")

print(res.fetchall())

res = cur.execute("""
    SELECT MIN(vote_average), title FROM movies
""")

print(res.fetchall())

[(13752, 'Inception')]
[(0.0, 'Black Water Transit')]

Aliases¶

An alias is a temporary name given to a column or table in a query. Aliases make query results easier to read and allow you to reference columns or tables with shorter or clearer names.

Aliases are created using the AS keyword and only apply to the result of the query.

In [30]:
res = cur.execute("""
    SELECT COUNT(*) AS total_movies, 
    AVG(budget) AS average_budget
    FROM movies
""")

print([col[0] for col in res.description])

print(res.fetchall())

['total_movies', 'average_budget']
[(4773, 29224612.72155877)]

Filtering Results¶

WHERE¶

The WHERE clause is used to filter rows in a query based on specified conditions.

Only rows that satisfy the condition in the WHERE clause are included in the query results.

Common operators include comparison operators (such as =, !=, <, >, <=, >=) and logical operators (such as AND, OR, and NOT) that combine or modify conditions.

In [31]:
res = cur.execute("""
    SELECT title, revenue FROM movies 
    WHERE revenue > 1000000000
    LIMIT 5
""")

res.fetchall()

LIKE¶

The LIKE operator is used in a WHERE clause to search for patterns in text values.

It often uses wildcard characters such as %, which represents any sequence of characters, and _, which represents a single character.

IN¶

The IN operator is used in a WHERE clause to check whether a value matches any value in a specified list.

It allows you to test a column against multiple possible values without writing multiple conditions.

AND and OR¶

The AND operator combines multiple conditions and returns rows only when all conditions are true.

The OR operator combines multiple conditions and returns rows when at least one condition is true.

In [34]:

    SELECT title, revenue, popularity FROM movies
    WHERE (revenue > 500000000 AND popularity < 10)
    OR (revenue < 100000000 AND popularity > 100)
""")

res.fetchall()

[('Fast Five', 626137675, 7),
 ('The Shawshank Redemption', 28341469, 136),
 ('The Godfather: Part II', 47542841, 105),

NULL¶

A NULL value represents missing or unknown data in a table.

Because NULL is not equal to any value, it must be checked using the IS NULL or IS NOT NULL conditions in a WHERE clause.

Grouping, Summarizing, and Sorting¶

GROUP BY¶

The GROUP BY clause is used to group rows that have the same values in specified columns.

It is commonly used with aggregate functions to calculate summaries for each group.

In [36]:
res = cur.execute("""
    SELECT director_id, SUM(revenue) AS total_revenue
    FROM movies
    GROUP BY director_id
    LIMIT 5
""")

res.fetchall()
Out[36]:
[(4762, 5883569439),
 (4763, 3177013357),
 (4764, 2713548809),
 (4765, 4227483234),
 (4766, 2109045355)]

HAVING¶

The HAVING clause is used to filter groups created by a GROUP BY clause.

It is typically used with aggregate functions to restrict which grouped results are included in the final output.

In [37]:
res = cur.execute("""
    SELECT director_id, SUM(revenue) AS total_revenue
    FROM movies
    GROUP BY director_id
    HAVING SUM(revenue) > 5000000000
    LIMIT 5
""")

res.fetchall()
Out[37]:
[(4762, 5883569439),
 (4777, 6498642820),
 (4788, 5832524638),
 (4799, 9147393164)]

ORDER BY¶

The ORDER BY clause is used to sort the results of a query based on one or more columns.

Results can be sorted in ascending order (ASC) or descending order (DESC).

In [38]:
res = cur.execute("""
    SELECT director_id, SUM(revenue) AS total_revenue
    FROM movies
    GROUP BY director_id
    ORDER BY total_revenue DESC
    LIMIT 5
""")

res.fetchall()
Out[38]:
[(4799, 9147393164),
 (4777, 6498642820),
 (4762, 5883569439),
 (4788, 5832524638),
 (4765, 4227483234)]

Joining Tables¶

JOIN and ON¶

A JOIN combines rows from two tables into a single result.

The ON clause specifies how the tables are related by defining which columns should match (usually a foreign key to a primary key).

Key Idea¶

  • JOIN = combine tables
  • ON = define how rows match

LEFT JOIN¶

A LEFT JOIN combines rows from two tables by matching values in related columns.

All rows from the left table are returned, and matching rows from the right table are included when they exist. If there is no match, the result contains NULL values for the columns from the right table.

In [39]:
res = cur.execute("""
    SELECT directors.id, directors.name, movies.title, movies.revenue FROM directors
    LEFT JOIN movies
    ON directors.id = movies.director_id
    LIMIT 5
""")

pprint(res.fetchall())

[(4762, 'James Cameron', 'The Terminator', 78371200),
 (4762, 'James Cameron', 'The Abyss', 90000098),
 (4762, 'James Cameron', 'Aliens', 183316455),
 (4762, 'James Cameron', 'True Lies', 378882411),
 (4762, 'James Cameron', 'Terminator 2: Judgment Day', 520000000)]

INNER JOIN¶

An INNER JOIN combines rows from two tables by matching values in related columns.

Only rows with matching values in both tables are included in the result.

OUTER JOIN¶

An OUTER JOIN returns rows that have matching values in both tables as well as rows that do not have matches.

When no match exists, the result includes NULL values for the columns from the table without a matching row.

Full Query Example¶

In [40]:
res = cur.execute("""
    SELECT directors.id, 
        directors.name, 
        SUM(movies.revenue) AS total_revenue
    FROM directors
    LEFT JOIN movies
    ON directors.id = movies.director_id
    WHERE movies.vote_average > 7
        AND movies.vote_count > 4000
        AND movies.tagline LIKE "%adventure%"
    GROUP BY directors.id
    HAVING SUM(movies.revenue) > 500000000 /* must come after GROUP BY */
    ORDER BY total_revenue DESC
""")

print(res.fetchall())
